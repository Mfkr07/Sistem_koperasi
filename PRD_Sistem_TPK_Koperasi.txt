================================================================================
1. RINGKASAN PRODUK
================================================================================

Aplikasi desktop offline berbasis Flutter untuk Windows yang digunakan oleh
satu orang pengurus koperasi sawit (TPK) guna mencatat seluruh transaksi
timbangan harian, mengelola pinjaman anggota, mencetak struk thermal, serta
menghasilkan laporan dan rekap yang dapat diekspor ke Excel, PDF, USB, maupun
folder lokal.

Aplikasi berjalan sepenuhnya tanpa koneksi internet. Seluruh data tersimpan
lokal menggunakan SQLite. Tidak ada mekanisme login; aplikasi langsung terbuka
dan siap digunakan oleh pengurus.


================================================================================
2. LATAR BELAKANG & KONTEKS BISNIS
================================================================================

Koperasi ini mengelola kegiatan timbang sawit harian yang melibatkan lebih dari
115 petani anggota yang dibagi ke dalam dua koordinator (Alek dan Yudi).
Setiap hari timbang, pengurus mencatat berat sawit tiap petani, menghitung
potongan (administrasi, transportasi, pinjaman), dan menyetor selisihnya kepada
petani. Di akhir sesi, pengurus juga mencatat pengeluaran seperti fee tenaga
kerja, biaya truck, KUD, jaga malam, dan kas TPK.

Selama ini pencatatan dilakukan secara manual di Excel dengan format flat
(satu baris per petani) yang rentan kesalahan hitung, tidak memiliki histori
pinjaman yang terstruktur, dan tidak dapat mencetak struk langsung untuk
petani.

Sistem ini dibuat untuk menggantikan proses manual tersebut dengan aplikasi
yang terstruktur, akurat, dan mampu mencetak struk thermal pada setiap
transaksi.


================================================================================
3. TUJUAN & RUANG LINGKUP
================================================================================

Tujuan:
- Mendigitalisasi pencatatan timbangan harian secara akurat dan cepat.
- Mengelola saldo pinjaman anggota dengan histori yang jelas.
- Mencetak struk thermal per transaksi sebagai bukti bagi petani.
- Menghasilkan rekap dan laporan harian yang dapat diekspor.
- Menyediakan backup data yang mudah ke USB atau folder.

Dalam Ruang Lingkup:
- Pencatatan sesi timbang harian.
- Manajemen data master anggota dan koordinator.
- Manajemen pinjaman anggota (tambah, potong, histori).
- Kalkulasi otomatis semua nilai turunan (ADM, TRS, HARGA, DISETOR).
- Cetak struk thermal per transaksi (USB dan Bluetooth).
- Cetak ulang struk berdasarkan nomor struk.
- Pencatatan pengeluaran sesi (KUD, fee, truck, dll).
- Rekap harian dan laporan per anggota.
- Export laporan ke Excel (.xlsx) dan PDF.
- Backup database ke USB atau folder lokal.
- Penanganan kepemilikan lahan bersama (dua nama per transaksi).
- Pembatalan (void) transaksi dalam sesi yang sama.

Di Luar Ruang Lingkup:
- Sinkronisasi cloud atau akses jaringan.
- Manajemen multi-pengguna atau sistem hak akses.
- Akuntansi lanjutan atau pembukuan koperasi penuh.
- Aplikasi mobile.

6. FITUR UTAMA (FEATURE LIST)
================================================================================

F-01  Manajemen Data Anggota
      Tambah, edit, nonaktifkan data petani anggota beserta koordinator
      dan jenis angkutan defaultnya.

F-02  Buka dan Tutup Sesi Timbang
      Membuat sesi harian baru dengan tanggal, koordinator, harga per Kg,
      tarif ADM, dan tarif TRS. Menutup sesi setelah semua transaksi selesai.

F-03  Input Transaksi Timbangan
      Mencatat berat sawit per petani dengan kalkulasi otomatis semua
      nilai turunan secara real-time.

F-04  Manajemen Pinjaman Anggota
      Mencatat pinjaman baru dan memotong saldo pinjaman secara otomatis
      saat transaksi timbang. Status LUNAS otomatis saat saldo nol.

F-05  Cetak Struk Thermal
      Mencetak struk per transaksi ke printer thermal (USB atau Bluetooth,
      58mm atau 80mm). Mendukung cetak ulang berdasarkan nomor struk.

F-06  Pencatatan Pengeluaran Sesi
      Mencatat semua biaya operasional per sesi: KUD, fee tenaga kerja,
      truck, jaga malam, kas TPK, dan lain-lain.

F-07  Rekap dan Laporan Harian
      Menampilkan ringkasan sesi: total tonase, total bruto, total potongan,
      total disetor, total pengeluaran, dan saldo kas bersih.

F-08  Laporan Per Anggota
      Menampilkan histori timbang, akumulasi setoran, dan riwayat pinjaman
      untuk satu anggota tertentu.

F-09  Rekap Global Multi-Sesi
      Menampilkan akumulasi tonase dan nilai per anggota lintas sesi dalam
      rentang tanggal tertentu.

F-10  Export Laporan
      Mengekspor laporan ke format Excel (.xlsx) dan PDF. Menyimpan ke
      folder yang dipilih pengguna atau langsung ke USB.

F-11  Backup Database
      Menyalin file database SQLite ke folder atau USB yang dipilih
      pengguna. Backup otomatis dipicu saat sesi ditutup.

F-12  Kepemilikan Bersama
      Mendukung transaksi dengan dua pemilik lahan (misal: NANDA/ASWADI)
      dengan pembagian porsi persentase dan struk terpisah per pemilik.

F-13  Pembatalan Transaksi
      Membatalkan (void) transaksi dalam sesi yang sama dengan rollback
      saldo pinjaman dan cetak struk pembatalan.

F-14  Pengaturan Aplikasi
      Mengatur nama koperasi, data koordinator, dan path default backup.

7. FLOW SISTEM
================================================================================

------------------------------------------------------------------------
7.1  SIKLUS OPERASIONAL HARIAN (OVERVIEW)
------------------------------------------------------------------------

Setiap hari timbang, pengurus menjalankan tiga fase secara berurutan:

    [FASE A: BUKA SESI]
           |
           v
    [FASE B: INPUT TIMBANGAN] <-- diulang untuk setiap petani
           |
           v
    [FASE C: TUTUP SESI & REKAP]

Di luar siklus harian, pengurus dapat sewaktu-waktu menjalankan:
- Manajemen Pinjaman (Flow D)
- Cetak / Export Laporan (Flow E)
- Manajemen Anggota
- Pengaturan

------------------------------------------------------------------------
7.2  FLOW A: BUKA SESI TIMBANG
------------------------------------------------------------------------

Dilakukan SATU KALI sebelum mulai mencatat timbangan hari itu.

  LANGKAH 1 - Buka Aplikasi
    Pengurus membuka aplikasi. Layar utama tampil.
    Sistem cek apakah ada sesi BUKA untuk hari ini.
      -> Jika ada: tampilkan notifikasi "Sesi hari ini sudah aktif",
         langsung masuk ke form timbang.
      -> Jika tidak ada: tampilkan tombol "Buka Sesi Baru".

  LANGKAH 2 - Buat Sesi Baru
    Pengurus menekan "Buka Sesi Baru". Form sesi tampil dengan:
    - Tanggal (auto-isi tanggal hari ini, bisa diubah)
    - Pilih koordinator: Alek / Yudi
    - Harga per Kg (default dari sesi terakhir, bisa diubah)
    - Tarif ADM per Kg (default dari sesi terakhir)
    - Tarif TRS Dusun per Kg (default dari sesi terakhir)
    - Tarif TRS Ibol per Kg (default dari sesi terakhir)

  LANGKAH 3 - Konfirmasi & Simpan
    Pengurus menekan "Buka Sesi".
    Sistem INSERT ke tabel sesi_timbang dengan status = BUKA.
    Layar berpindah ke form input timbangan.

    DB yang dipengaruhi: sesi_timbang (INSERT)

------------------------------------------------------------------------
7.3  FLOW B: INPUT TIMBANGAN PER PETANI
------------------------------------------------------------------------

Diulang untuk SETIAP petani yang menimbang hari itu.

  LANGKAH 1 - Cari dan Pilih Anggota
    Pengurus mengetik nama atau memilih dari daftar dropdown.
    Sistem menampilkan:
    - Nama lengkap anggota
    - Koordinator yang menaungi
    - Jenis angkutan default (SENDIRI / DUSUN / IBOL)
    - Saldo pinjaman aktif (jumlah total dari semua pinjaman AKTIF)
    Jika anggota sudah pernah timbang di sesi ini, tampilkan peringatan
    "Anggota sudah timbang hari ini. Lanjutkan?"

    DB yang diakses: anggota (SELECT), pinjaman (SELECT SUM saldo_sisa)

  LANGKAH 2 - Input Berat
    Pengurus memasukkan angka berat (Kg) dari hasil timbangan.
    Saat nilai berubah, sistem langsung hitung dan tampilkan preview:
    - HARGA BRUTO  = berat_kg x harga_per_kg
    - BIAYA ADM    = berat_kg x tarif_adm_per_kg

  LANGKAH 3 - Konfirmasi Jenis Angkutan
    Jenis angkutan terisi otomatis dari data default anggota.
    Pengurus dapat mengubah jika berbeda hari ini.
    Sistem hitung otomatis:
    - BIAYA TRS    = berat_kg x tarif_trs (sesuai jenis angkutan)
    - SENDIRI      = Rp 0

  LANGKAH 4 - Input Potongan Pinjaman
    Jika saldo pinjaman aktif > 0:
      - Field pinjaman aktif dengan nilai default 0.
      - Pengurus input nominal yang akan dipotong.
      - Sistem validasi: nominal dipotong tidak boleh melebihi saldo aktif.
      - Jika melebihi, tampilkan pesan error dan blokir.
    Jika saldo pinjaman = 0:
      - Field pinjaman dinonaktifkan (readonly, nilai 0).

  LANGKAH 5 - Kalkulasi Final (Otomatis, Real-time)
    Sistem hitung dan tampilkan semua nilai secara real-time:
    - TOTAL POTONGAN  = ADM + TRS + pinjaman_dipotong
    - JUMLAH DISETOR  = HARGA BRUTO - TOTAL POTONGAN
    Validasi: JUMLAH DISETOR tidak boleh bernilai negatif.
    Jika negatif, tampilkan peringatan merah dan blokir tombol SIMPAN.

  LANGKAH 6 - Konfirmasi dan Simpan
    Pengurus menekan "SIMPAN".
    Sistem melakukan:
    a. INSERT ke tabel transaksi dengan berat_kg dan pinjaman_dipotong.
       (Nilai ADM, TRS, HARGA, DISETOR TIDAK disimpan - dihitung saat query)
    b. Generate nomor struk: STR-[YYYYMMDD]-[urutan 3 digit], misal
       STR-20260401-042.
    c. UPDATE saldo_sisa pinjaman:
       saldo_baru = saldo_lama - pinjaman_dipotong
       Jika saldo_baru = 0, UPDATE status pinjaman menjadi LUNAS.
    d. Tampilkan konfirmasi sukses.

    DB yang dipengaruhi: transaksi (INSERT), pinjaman (UPDATE)

  LANGKAH 7 - Cetak Struk Thermal
    Setelah simpan, muncul dialog "Cetak struk sekarang?"
    Jika ya:
      - Sistem render template struk ESC/POS.
      - Kirim perintah ke printer yang terhubung (USB atau Bluetooth).
      - UPDATE transaksi.sudah_cetak = true dan isi waktu_cetak.
    Jika tidak / printer tidak terhubung:
      - Data tersimpan, transaksi.sudah_cetak tetap false.
      - Dapat dicetak ulang nanti dari menu "Cetak Ulang".

  LANGKAH 8 - Lanjut Petani Berikutnya
    Form direset, kembali ke Langkah 1 untuk petani berikutnya.

  [KASUS KHUSUS: KEPEMILIKAN BERSAMA]
    Jika satu bidang lahan dimiliki dua petani (misal: NANDA/ASWADI):
    - Pengurus centang "Kepemilikan Bersama" di form.
    - Pilih dua anggota dan isi persentase masing-masing (total harus 100%).
    - Sistem INSERT satu transaksi utama dan dua baris kepemilikan_bersama.
    - Sistem cetak dua struk terpisah (satu per pemilik) dengan nilai
      disetor masing-masing sesuai porsi persentase.

------------------------------------------------------------------------
7.4  FLOW C: TUTUP SESI & REKAP
------------------------------------------------------------------------

Dilakukan SATU KALI setelah semua petani selesai ditimbang.

  LANGKAH 1 - Input Pengeluaran Sesi
    Sebelum menutup sesi, pengurus mencatat semua pengeluaran:
    - KUD
    - Iuran anggota
    - Fee tenaga kerja (per nama penerima dan jumlah)
    - Biaya jaga malam
    - Biaya truck / angkutan
    - Kas TPK
    - Lain-lain
    Setiap item INSERT ke tabel pengeluaran dengan sesi_id yang aktif.

  LANGKAH 2 - Lihat Rekap Harian
    Sistem otomatis hitung dan tampilkan ringkasan sesi:
    - Total anggota yang timbang
    - Total tonase (SUM berat_kg)
    - Total harga bruto (SUM berat_kg x harga_per_kg)
    - Total biaya ADM (SUM berat_kg x tarif_adm)
    - Total biaya TRS (SUM berat_kg x tarif_trs)
    - Total pinjaman dipotong (SUM pinjaman_dipotong)
    - Total disetor ke petani
    - Total pengeluaran sesi (SUM pengeluaran.jumlah)
    - Saldo kas bersih

  LANGKAH 3 - Cetak atau Export Rekap
    Pilihan output rekap harian:
    a. Cetak ke printer thermal (struk panjang)
    b. Export ke Excel (.xlsx) - simpan ke folder atau USB
    c. Export ke PDF - simpan ke folder atau USB
    d. Hanya tampil di layar

  LANGKAH 4 - Tutup Sesi
    Pengurus menekan "Tutup Sesi".
    Konfirmasi dialog: "Yakin menutup sesi? Transaksi baru tidak dapat
    ditambahkan setelah sesi ditutup."
    Jika dikonfirmasi:
    - UPDATE sesi_timbang.status = TUTUP.
    - Sistem otomatis backup file SQLite ke path backup yang telah
      dikonfigurasi (USB atau folder). Nama file: db_YYYYMMDD.sqlite.
    - Tampilkan notifikasi "Sesi ditutup. Backup berhasil disimpan di [path]."

    DB yang dipengaruhi: sesi_timbang (UPDATE status)

------------------------------------------------------------------------
7.5  FLOW D: MANAJEMEN PINJAMAN
------------------------------------------------------------------------

Dapat dilakukan kapan saja, tidak bergantung pada sesi timbang.

  ALUR D1: TAMBAH PINJAMAN BARU
    1. Pengurus buka menu Pinjaman.
    2. Pilih anggota dari daftar.
    3. Sistem tampilkan saldo pinjaman aktif yang sudah ada.
    4. Pengurus input jumlah pokok pinjaman baru dan keterangan.
    5. Konfirmasi dan simpan.
    6. Sistem INSERT ke tabel pinjaman dengan status = AKTIF dan
       saldo_sisa = jumlah_pokok.

    DB yang dipengaruhi: pinjaman (INSERT)

  ALUR D2: POTONGAN PINJAMAN (otomatis saat timbang - lihat Flow B)

  ALUR D3: LIHAT HISTORI PINJAMAN ANGGOTA
    1. Pilih anggota.
    2. Sistem tampilkan daftar semua pinjaman (AKTIF dan LUNAS).
    3. Tampilkan detail: tanggal pinjam, pokok, saldo sisa, status.

  STATE MACHINE PINJAMAN:
    [tidak ada] --[tambah pinjaman baru]--> [AKTIF]
    [AKTIF]     --[dipotong sebagian]-----> [AKTIF] (saldo berkurang)
    [AKTIF]     --[saldo = 0]-------------> [LUNAS] (otomatis)
    [LUNAS]     --[tidak bisa diubah]-----> [LUNAS] (read-only histori)

  Catatan: Satu anggota bisa memiliki lebih dari satu pinjaman AKTIF.
  Pemotongan dilakukan secara FIFO (pinjaman terlama dipotong lebih dulu)
  secara otomatis, atau pengurus dapat memilih pinjaman mana yang dipotong.

------------------------------------------------------------------------
7.6  FLOW E: LAPORAN & EXPORT
------------------------------------------------------------------------

Jenis laporan yang tersedia:

  E1 - STRUK TRANSAKSI
    Trigger  : Otomatis setelah simpan timbangan
    Format   : Thermal 58mm atau 80mm (ESC/POS)
    Tujuan   : Diberikan langsung ke petani

  E2 - REKAP HARIAN
    Trigger  : Saat tutup sesi atau request manual
    Format   : Thermal (cetak), Excel (.xlsx), PDF
    Tujuan   : Arsip pengurus dan koperasi
    Isi      : Semua transaksi sesi, total, pengeluaran, saldo kas

  E3 - LAPORAN PER ANGGOTA
    Trigger  : Request manual, pilih anggota dan rentang tanggal
    Format   : Layar, PDF, Excel
    Tujuan   : Rekap histori petani, kontrol pinjaman
    Isi      : Histori timbang, akumulasi setoran, riwayat pinjaman

  E4 - LAPORAN PINJAMAN
    Trigger  : Request manual
    Format   : Layar, Excel, PDF
    Tujuan   : Monitoring piutang koperasi
    Isi      : Semua pinjaman AKTIF beserta saldo sisa per anggota

  E5 - LAPORAN PENGELUARAN
    Trigger  : Request manual atau saat tutup sesi
    Format   : Layar, PDF
    Tujuan   : Keuangan operasional koperasi
    Isi      : Detail biaya per kategori per sesi

  E6 - REKAP GLOBAL MULTI-SESI
    Trigger  : Request manual, pilih rentang tanggal
    Format   : Layar, Excel
    Tujuan   : Perbandingan kinerja lintas periode
    Isi      : Akumulasi tonase dan nilai per petani

  ALUR EXPORT:
    1. Pengurus pilih jenis laporan dan parameter (tanggal, anggota, dll).
    2. Sistem generate data dari query SQLite.
    3. Pengurus pilih format: Excel / PDF / Cetak.
    4. Dialog "Simpan ke" muncul: pilih folder atau drive USB.
    5. Sistem simpan file dengan nama otomatis:
       - Excel: Rekap_Harian_YYYYMMDD.xlsx
       - PDF  : Rekap_Harian_YYYYMMDD.pdf


================================================================================
8. SPESIFIKASI FUNGSIONAL
================================================================================

------------------------------------------------------------------------
8.1  MODUL ANGGOTA
------------------------------------------------------------------------

FR-ANK-01  Tambah anggota baru dengan field: nama, koordinator, jenis
           angkutan default (SENDIRI/DUSUN/IBOL), nomor HP (opsional).
FR-ANK-02  Edit data anggota yang sudah ada.
FR-ANK-03  Nonaktifkan anggota (soft delete, data historis tetap tersimpan).
FR-ANK-04  Cari anggota by nama (partial match, case-insensitive).
FR-ANK-05  Tampilkan daftar anggota dengan filter: semua / aktif / nonaktif.
FR-ANK-06  Tampilkan saldo pinjaman aktif di daftar anggota.
FR-ANK-07  Mendukung dua nama dalam satu anggota untuk kepemilikan bersama.

------------------------------------------------------------------------
8.2  MODUL SESI TIMBANG
------------------------------------------------------------------------

FR-SESI-01  Buat sesi baru dengan tanggal, koordinator, harga per Kg,
            tarif ADM, tarif TRS Dusun, tarif TRS Ibol.
FR-SESI-02  Nilai default tarif dan harga diambil dari sesi terakhir.
FR-SESI-03  Hanya boleh ada satu sesi berstatus BUKA pada satu waktu.
FR-SESI-04  Tutup sesi mengubah status menjadi TUTUP dan memicu backup otomatis.
FR-SESI-05  Setelah sesi TUTUP, tidak dapat menambah transaksi ke sesi tersebut.
FR-SESI-06  Tampilkan histori sesi yang lalu (tanggal, total tonase, status).

------------------------------------------------------------------------
8.3  MODUL TRANSAKSI
------------------------------------------------------------------------

FR-TRX-01  Input transaksi hanya bisa dilakukan dalam sesi berstatus BUKA.
FR-TRX-02  Pilih anggota dengan fitur pencarian.
FR-TRX-03  Input berat Kg (numerik, tidak boleh nol atau negatif).
FR-TRX-04  Preview kalkulasi real-time saat pengguna mengetik berat.
FR-TRX-05  Pilih jenis angkutan (default dari data anggota, bisa diubah).
FR-TRX-06  Input pinjaman dipotong (default 0, tidak boleh melebihi saldo aktif).
FR-TRX-07  Tombol SIMPAN hanya aktif jika semua validasi lulus.
FR-TRX-08  Nilai ADM, TRS, HARGA BRUTO, dan JUMLAH DISETOR tidak disimpan
           di database; selalu dihitung ulang dari data mentah saat diperlukan.
FR-TRX-09  Generate nomor struk unik format STR-YYYYMMDD-NNN.
FR-TRX-10  Tandai transaksi sudah_cetak setelah printer berhasil mencetak.
FR-TRX-11  Tampilkan daftar transaksi belum cetak untuk cetak ulang.
FR-TRX-12  Void (batalkan) transaksi dalam sesi yang sama dengan konfirmasi.
           Transaksi di-flag void=true, tidak dihapus dari database.
           Saldo pinjaman di-rollback sesuai nilai pinjaman_dipotong semula.

------------------------------------------------------------------------
8.4  MODUL PINJAMAN
------------------------------------------------------------------------

FR-PIN-01  Tambah pinjaman baru kapan saja (tidak perlu sesi aktif).
FR-PIN-02  Tampilkan total saldo aktif anggota saat dipilih di form timbang.
FR-PIN-03  Potongan pinjaman saat timbang tidak boleh melebihi saldo aktif.
FR-PIN-04  Update saldo_sisa otomatis setiap ada potongan.
FR-PIN-05  Status otomatis berubah ke LUNAS saat saldo_sisa = 0.
FR-PIN-06  Pinjaman LUNAS bersifat read-only, hanya untuk histori.
FR-PIN-07  Tampilkan histori semua pinjaman per anggota.
FR-PIN-08  Pemotongan FIFO secara default; pengurus dapat memilih manual.

------------------------------------------------------------------------
8.5  MODUL PENGELUARAN
------------------------------------------------------------------------

FR-PGL-01  Tambah pengeluaran per sesi dengan kategori:
           KUD, TENAGA_KERJA, TRUCK, JAGA_MALAM, KAS, LAIN.
FR-PGL-02  Input nama penerima dan jumlah untuk setiap item pengeluaran.

9. ATURAN BISNIS & FORMULA KALKULASI
================================================================================

Semua nilai berikut bersifat DERIVED (turunan) dan TIDAK DISIMPAN di database.
Nilai dihitung ulang setiap kali diperlukan (tampilan, laporan, struk).

------------------------------------------------------------------------
FORMULA KALKULASI TRANSAKSI
------------------------------------------------------------------------

  HARGA BRUTO      = berat_kg x harga_per_kg
  BIAYA ADM        = berat_kg x tarif_adm_per_kg
  BIAYA TRS        = berat_kg x tarif_trs
                     - Jenis DUSUN  : tarif_trs_dusun (dari sesi)
                     - Jenis IBOL   : tarif_trs_ibol (dari sesi)
                     - Jenis SENDIRI: Rp 0
  TOTAL POTONGAN   = BIAYA ADM + BIAYA TRS + pinjaman_dipotong
  JUMLAH DISETOR   = HARGA BRUTO - TOTAL POTONGAN

  Contoh (data aktual dari 01_APRIL_2026.xls):
    Anggota    : RASYID
    Berat      : 223 Kg
    Harga/Kg   : Rp 16.900
    Tarif ADM  : Rp 100/Kg
    Tarif TRS  : Rp 350/Kg (DUSUN)
    Pinjaman   : Rp 300.000

    Harga Bruto    = 223 x 16.900          = Rp 3.768.700
    ADM            = 223 x 100             = Rp    22.300
    TRS            = 223 x 350             = Rp    78.050
    Total Potongan = 22.300 + 78.050 + 300.000 = Rp 400.350
    Disetor        = 3.768.700 - 400.350   = Rp 3.368.350

------------------------------------------------------------------------
UPDATE SALDO PINJAMAN
------------------------------------------------------------------------

  saldo_baru = saldo_lama - pinjaman_dipotong
  Jika saldo_baru = 0 maka status = LUNAS

------------------------------------------------------------------------
ATURAN BISNIS
------------------------------------------------------------------------

BR-01  Transaksi baru hanya bisa dibuat dalam sesi berstatus BUKA.
BR-02  Nilai berat_kg harus lebih besar dari 0.
BR-03  Nilai pinjaman_dipotong tidak boleh melebihi saldo_sisa aktif anggota.
BR-04  Nilai JUMLAH DISETOR tidak boleh negatif.
BR-05  Satu sesi hanya boleh ada satu yang berstatus BUKA pada satu waktu.
BR-06  Setelah sesi TUTUP, transaksi di dalamnya tidak bisa ditambah atau
       diedit, hanya bisa void dengan konfirmasi.
BR-07  Void transaksi hanya bisa dilakukan dalam sesi yang sama (BUKA).
BR-08  Porsi kepemilikan bersama harus berjumlah tepat 100%.
BR-09  Pinjaman yang sudah LUNAS tidak bisa dimodifikasi.
BR-10  Data anggota yang dinonaktifkan tetap muncul di histori transaksi lama.

11. SKEMA DATABASE (NORMALISASI 3NF)
================================================================================

Semua tabel berada dalam satu file SQLite lokal.
Nilai derived (ADM, TRS, HARGA, DISETOR) tidak disimpan; dihitung saat query.

------------------------------------------------------------------------
TABEL: koordinator
------------------------------------------------------------------------
  Kolom           Tipe           Constraint        Keterangan
  ----------------------------------------------------------------------
  koordinator_id  TEXT           PRIMARY KEY       Contoh: KOOR-ALEK
  nama            TEXT           NOT NULL          Nama koordinator
  catatan         TEXT           nullable          Keterangan tambahan
  created_at      TEXT           NOT NULL          Format ISO 8601

------------------------------------------------------------------------
TABEL: anggota
------------------------------------------------------------------------
  Kolom           Tipe           Constraint        Keterangan
  ----------------------------------------------------------------------
  anggota_id      TEXT           PRIMARY KEY       Contoh: ANK-001
  koordinator_id  TEXT           FK -> koordinator
  nama            TEXT           NOT NULL          Nama lengkap petani
  tipe_angkutan   TEXT           NOT NULL          SENDIRI / DUSUN / IBOL
  no_hp           TEXT           nullable
  status_aktif    INTEGER        DEFAULT 1         1 = aktif, 0 = nonaktif
  created_at      TEXT           NOT NULL

------------------------------------------------------------------------
TABEL: sesi_timbang
------------------------------------------------------------------------
  Kolom               Tipe      Constraint        Keterangan
  ----------------------------------------------------------------------
  sesi_id             TEXT      PRIMARY KEY       Contoh: SESI20260401
  koordinator_id      TEXT      FK -> koordinator
  tanggal             TEXT      NOT NULL          Format YYYY-MM-DD
  harga_per_kg        INTEGER   NOT NULL          Harga sawit hari ini
  tarif_adm_per_kg    INTEGER   NOT NULL          Tarif ADM per Kg
  tarif_trs_dusun     INTEGER   NOT NULL          Tarif TRS jenis Dusun
  tarif_trs_ibol      INTEGER   NOT NULL          Tarif TRS jenis Ibol
  status              TEXT      NOT NULL          BUKA / TUTUP
  catatan             TEXT      nullable
  created_at          TEXT      NOT NULL
  closed_at           TEXT      nullable          Waktu sesi ditutup

------------------------------------------------------------------------
TABEL: transaksi
------------------------------------------------------------------------
  Kolom               Tipe      Constraint        Keterangan
  ----------------------------------------------------------------------
  transaksi_id        TEXT      PRIMARY KEY       Auto-generate
  sesi_id             TEXT      FK -> sesi_timbang
  anggota_id          TEXT      FK -> anggota
  berat_kg            INTEGER   NOT NULL          Nilai dari timbangan
  pinjaman_dipotong   INTEGER   DEFAULT 0         Potongan hutang
  no_struk            TEXT      UNIQUE            STR-YYYYMMDD-NNN
  sudah_cetak         INTEGER   DEFAULT 0         0 = belum, 1 = sudah
  waktu_cetak         TEXT      nullable
  waktu_input         TEXT      NOT NULL
  is_void             INTEGER   DEFAULT 0         0 = valid, 1 = void

  Catatan: ADM, TRS, HARGA, TOTAL POTONGAN, DISETOR tidak disimpan.
  Semua dihitung dari berat_kg + data tarif di sesi_timbang.

------------------------------------------------------------------------
TABEL: pinjaman
------------------------------------------------------------------------
  Kolom           Tipe           Constraint        Keterangan
  ----------------------------------------------------------------------
  pinjaman_id     TEXT           PRIMARY KEY
  anggota_id      TEXT           FK -> anggota
  tanggal_pinjam  TEXT           NOT NULL
  jumlah_pokok    INTEGER        NOT NULL          Nilai pinjaman awal
  saldo_sisa      INTEGER        NOT NULL          Sisa belum lunas
  status          TEXT           NOT NULL          AKTIF / LUNAS
  keterangan      TEXT           nullable
  created_at      TEXT           NOT NULL

------------------------------------------------------------------------
TABEL: pengeluaran
------------------------------------------------------------------------
  Kolom             Tipe         Constraint        Keterangan
  ----------------------------------------------------------------------
  pengeluaran_id    TEXT         PRIMARY KEY
  sesi_id           TEXT         FK -> sesi_timbang
  kategori          TEXT         NOT NULL          KUD / TENAGA_KERJA /
                                                   TRUCK / JAGA_MALAM /
                                                   KAS / LAIN
  nama_penerima     TEXT         nullable          Untuk TENAGA_KERJA
  jumlah            INTEGER      NOT NULL
  keterangan        TEXT         nullable
  created_at        TEXT         NOT NULL

------------------------------------------------------------------------
TABEL: kepemilikan_bersama
------------------------------------------------------------------------
  Kolom           Tipe           Constraint        Keterangan
  ----------------------------------------------------------------------
  id              INTEGER        PRIMARY KEY AUTOINCREMENT
  transaksi_id    TEXT           FK -> transaksi
  anggota_id      TEXT           FK -> anggota
  porsi_persen    REAL           NOT NULL          Contoh: 50.00
  catatan         TEXT           nullable          Contoh: NANDA/ASWADI

------------------------------------------------------------------------
RELASI ANTAR TABEL
------------------------------------------------------------------------

  koordinator    (1) ----> (N)  anggota
  koordinator    (1) ----> (N)  sesi_timbang
  sesi_timbang   (1) ----> (N)  transaksi
  sesi_timbang   (1) ----> (N)  pengeluaran
  anggota        (1) ----> (N)  transaksi
  anggota        (1) ----> (N)  pinjaman
  transaksi      (1) ----> (N)  kepemilikan_bersama  [opsional]

  ================================

  Tgl Batal     : DD/MM/YYYY HH:MM
  Anggota       : [NAMA]

  ================================
  Transaksi ini dibatalkan.

PERFORMA

CATATAN TEKNIS PENTING
------------------------------------------------------------------------

  1. Semua query aggregate (SUM, COUNT) untuk laporan dan rekap
     dieksekusi langsung di SQLite untuk performa optimal.

  2. Nilai ADM, TRS, HARGA BRUTO, dan JUMLAH DISETOR TIDAK pernah
     di-INSERT ke database. Selalu dihitung ulang di layer service
     menggunakan data berat_kg dan tarif dari sesi_timbang.

  3. Untuk cetak struk via USB di Windows, gunakan pendekatan
     win32 API (CreateFile ke nama port printer) atau lewat
     Windows Spooler menggunakan RawPrinterHelper.

  4. File SQLite disimpan di folder yang bisa dipilih pengurus,
     disarankan default di Documents\TPK_Koperasi\data.db agar
     mudah ditemukan dan di-backup manual.

  5. Gunakan SQLite WAL mode (Write-Ahead Logging) untuk mengurangi
     risiko korupsi data jika aplikasi ditutup paksa.
